# prjct vs Other Tools

## Understanding the Landscape

Developer tools fall into two categories:

| Category | What they do | Examples |
|----------|--------------|----------|
| **Coding Agents** | Write code for you | OpenCode, Cursor, Copilot, Claude Code |
| **Workflow Tools** | Track progress & context | prjct, Jira, Linear, Notion |

**prjct is a workflow tool, not a coding agent.**

## prjct vs Coding Agents

### vs OpenCode.ai

| | OpenCode | prjct-cli |
|---|----------|-----------|
| **Purpose** | AI writes code | Track what you're building |
| **Replaces** | Your IDE/editor | Jira/Linear/Notion |
| **Model support** | 75+ providers | Claude-optimized |
| **Value** | Multi-provider, privacy | Memory, workflow, focus |

**Use together:** OpenCode writes code, prjct tracks progress.

### vs Cursor / Copilot

| | Cursor/Copilot | prjct-cli |
|---|----------------|-----------|
| **Purpose** | AI code completion | Developer workflow |
| **Category** | IDE extension | CLI tool |
| **Focus** | Writing code faster | Shipping features faster |

**Use together:** Cursor completes code, prjct tracks what you ship.

### vs Claude Code

| | Claude Code | prjct-cli |
|---|-------------|-----------|
| **Purpose** | AI coding agent | Workflow enhancement |
| **Relationship** | The engine | The dashboard |
| **Category** | AI agent | Productivity layer |

**prjct enhances Claude Code.** It adds:
- Persistent memory between sessions
- Domain-specific agents
- Task tracking (`p. task` -> `p. done` -> `p. ship`)
- Progress metrics

## prjct vs Workflow Tools

### vs Jira / Linear

| | Jira/Linear | prjct-cli |
|---|-------------|-----------|
| **Overhead** | Tickets, sprints, ceremonies | Zero - just `p. task` |
| **Data** | Cloud, team-centric | Local, developer-centric |
| **Focus** | Team coordination | Individual momentum |
| **Cost** | $7-14/user/month | Free |

**prjct is for builders who ship, not managers who plan.**

### vs Notion / Obsidian

| | Notion/Obsidian | prjct-cli |
|---|-----------------|-----------|
| **Purpose** | General notes | Developer workflow |
| **Integration** | Manual | Native Claude Code |
| **Context** | You remember | AI remembers |

**prjct is purpose-built for development workflows.**

## When to Use What

| Scenario | Tool |
|----------|------|
| Need AI to write code | OpenCode, Cursor, Claude Code |
| Need to track what you're building | **prjct** |
| Need team coordination | Jira, Linear |
| Need personal notes | Notion, Obsidian |
| Need developer momentum | **prjct** |

## The prjct Philosophy

1. **Zero ceremony** - No sprints, no story points, no standups
2. **Single focus** - One task at a time
3. **Ship-centric** - Track features shipped, not hours worked
4. **Local-first** - Your data, your machine
5. **Claude-native** - Designed for Claude Code

## Summary

**prjct doesn't compete with coding agents. It complements them.**

```
OpenCode/Cursor/Claude Code → Writes code
prjct-cli                   → Tracks progress, maintains context, keeps you focused
```

Use both together for maximum developer momentum.
