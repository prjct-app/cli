# 🔍 Análisis de Competencia - Copy & Posicionamiento

## Los 4 Referentes

### 1. OpenCode.ai
**Posicionamiento**: Open source AI coding agent

**Headlines**:
- No headline principal claro en landing
- Se enfoca en: "Bring Your Own Model"

**Enfoque**:
- Model agnostic (75+ providers)
- Privacy first
- Terminal-based
- Multi-session support

**Tono**: Técnico, developer-first, open source ethos

---

### 2. Warp.dev
**Posicionamiento**: The Agentic Development Environment

**Headlines**:
- "Warp 2.0 — World's First Agentic Development Environment"
- "A modern, AI‑powered terminal for high‑velocity development"

**Enfoque**:
- Terminal reinventado
- AI agents integrados
- Team collaboration
- Enterprise ready

**Tono**: Bold, aspiracional, "the future is here"

---

### 3. Cursor.com
**Posicionamiento**: The AI Code Editor

**Headlines**:
- "The AI Code Editor" (simple, directo)
- Se enfoca en autonomy slider concept

**Enfoque**:
- Fork de VS Code con AI nativo
- Tab completion inteligente
- Multi-model (OpenAI, Anthropic, Gemini, xAI)
- Codebase-aware

**Tono**: Minimalista, confidence tranquila, deja que el producto hable

---

### 4. Windsurf/Codeium
**Posicionamiento**: The first AI-native IDE

**Headlines**:
- "Tomorrow's editor, today."
- "The first agentic IDE, and then some."
- "Type less. Code more. Ship faster."

**Enfoque**:
- AI-native desde cero
- "Flow state" como diferenciador
- Agentic capabilities
- Free tier generoso

**Tono**: Aspiracional pero accesible, énfasis en "flow" y "magic"

## 🎯 Patrones Identificados

### Lo Que Hacen Todos

1. **Posicionamiento como "el primero"**
   - Warp: "World's First Agentic Development Environment"
   - Windsurf: "The first AI-native IDE"
   - Cursor: "The AI Code Editor" (implica definitivo)

2. **Palabras clave recurrentes**
   - "Agentic"
   - "Flow"
   - "Native"
   - "Ship faster"

3. **Énfasis en velocidad**
   - "high-velocity development"
   - "Ship faster"
   - "Code more"

4. **Multi-model como feature**
   - Todos mencionan soporte para múltiples LLMs

### Lo Que NO Hacen

1. **No muestran métricas específicas**
   - Ninguno dice "X tokens ahorrados"
   - No hay números concretos de eficiencia

2. **No explican el "cómo"**
   - Son muy aspiracionales
   - Poca profundidad técnica en landing

3. **No hablan de persistencia/memoria**
   - Nadie menciona que el contexto se pierde
   - No hablan de aprendizaje entre sesiones
